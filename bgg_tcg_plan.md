# BGG TCG (桌遊卡牌對戰) - 新版開發規劃

這份文件旨在規劃如何將現有的 Pocket Monster TCG 轉換為「BGG 桌遊卡牌對戰」，利用我們先前透過 SQLite 處理篩選出的 14,471 款桌遊資料，自動生成為可遊玩的實體卡牌。

---

## 🎲 1. 卡牌資料轉換邏輯

我們將透過 Python 腳本從 `bgg.db` 將桌遊資料轉換為遊戲可讀的 `cards.json`。每款桌遊將成為一張獨立的卡牌，數值計算公式如下：

### 📸 基礎屬性
* **圖片 (Image)**: 替換原本的 Emoji，改為從資料庫 `image` 欄位讀取真實的桌遊封面圖。
* **卡牌名稱 (Name)**: `name`。如果名稱過長，UI 上需要實作跑馬燈或自動縮小字體機制。

### ⚔️ 戰鬥數值計算
根據使用者的要求，具體的數值算式：
* **HP**: `50 * log10(rating_count)`
  * *說明*: 評價人數越多的熱門遊戲血量越高，使用對數 (log) 避免差異過大。
* **攻擊力 (ATK)**: `rating_avg * 10`
  * *說明*: 遊戲評價越高，攻擊力越強 。
* **速度 (SPD)**: `(min_playtime + max_playtime) / (2 * min_players)`
如果只存在`min_playtime`與`max_playtime`其中一個，則改成 
`(playtime) / (min_players)`
  * *說明*: 平均遊戲時間除以最小玩家數。此值在此公式下可能是「時間越長速度數值越大」，在多數 RPG 中速度通常代表敏捷，若要將重策遊戲視為「慢但重擊」或「快節奏遊戲速度快」，可能需要在 UI 取捨或加上反轉 (例如：上限值 - 算出來的值)。我們目前先嚴格照公式實作。
* **棄牌代價 (Discard)**: `floor(weight_avg)` (取下高斯，即無條件捨去至整數)
  * *說明*: BGG 的重度平均值 (1.0~5.0)，無條件捨去後會得到 1, 2, 3, 4, 5，完美符合棄牌代價的需求（越硬核的遊戲所需代價越大）。

### 🌟 稀有度系統 (Rarity)
根據卡牌在 overall 的 `rank` 排名來決定稀有度與對應顏色，顏色將直接影響卡牌的外框與背景特效：

| 稀有度 | Rank 區間 | 視覺主題色 | 邊框特效 |
| --- | --- | --- | --- |
| **UR** | 1 – 50 | 金色 | 閃爍流光特效，動態全息背景 |
| **SSR** | 51 – 200 | 紅色 | 強烈的紅色發光邊框 |
| **SR** | 201 – 800 | 紫色 | 紫色暗光，高雅的漸層質感 |
| **R** | 801 – 2500 | 藍色 | 藍色霓虹框線 |
| **UC** | 2501 – 6000 | 綠色 | 綠色金屬光澤 |
| **C** | 6001+ | 白色 | 基礎的灰白或大地色系 |

## 🖼️ 2. UI / UX 重新設計規劃

目前的 UI 是針對大 Emoji 與文字設計的，轉移到 BGG 桌遊封面圖後，需要進行全面改版以提升辨識度與遊玩體驗：

### 🃏 卡牌視覺設計
1. **滿版封面背景**: 放棄 Emoji 區塊，卡牌上半部（或整個背景）直上桌遊的 `image`（設定為 `object-fit: cover` 加上頂部微漸層黑底讓名字清楚）。
2. **數值 icon 化設計**:
   * 將 HP、ATK (⚔️)、SPD (⚡) 和棄牌 (🎴) 放在卡片底部，確保在各種花俏的桌遊封面上都能被看清楚。
3. **稀有度徽章**: 在右上角或左上角增加 UR, SSR 等小徽章。
4. **長名稱處理**: 桌遊名稱往往很長（例如 *Twilight Imperium: Fourth Edition*），可以設計為限定行高 (line-clamp) 或是在滑鼠懸停/點擊放大時顯示全名。

### 📱 介面操作改良
1. **抽卡特效 (Gacha)**: 抽卡時改為顯示卡背（可以設計一個統一的桌遊盒子卡背），翻開時會有對應稀有度顏色的光芒爆炸特效。
2. **戰鬥辨識**: 螢幕中間對戰的兩張卡牌要夠大，讓對戰的「兩款桌遊」的封面成為畫面的重心。
3. **防禦機制 (可選擴充)**: 既然有了 BGG 的重度，目前的代價是棄牌，或許未來可以把 `weight_avg` 當成防禦力，增加更深度的對戰策略。

## 🚀 3. 現有資源執行步驟 (How to Execute)

為了將原本的專案轉換為 BGG 版本，我們需要分三個步驟執行：

### 階段一：資料生成 (Python)
1. 在 `BGG` 資料夾中撰寫一個匯出腳本 `export_bgg_cards.py`。
2. 該腳本會連接 `bgg.db`，使用我們先前 `verify_db_stats.py` 中寫好的 SQL 篩選條件。
3. 讀出這 14,471 筆資料。
4. 在 Python 內對每筆資料計算 `hp`, `atk`, `spd`, `discard`, `rarity`。
5. 將結果匯出成一個龐大的 `cards.json`，並取代 `pocket-monster-tcg/cards.json`。

### 階段二：改寫前端資料結構與樣式 (HTML/CSS)
1. 進入 `pocket-monster-tcg/index.html`。
2. 修改 `createCardHTML()` 函數：
   * 將 `emoji` 替換為 `<img>` 標籤載入 `card.image`。
   * 為外層加上基於稀有度（`card.rarity`）的 CSS Class，用以顯示不同的卡框與顏色。
3. 更新 `CSS`：加入 UR, SSR, SR 等等級專屬的動畫與流光邊框樣式 (`box-shadow`, `border-color` 等)。

### 階段三：效能與機率調整 (Optimization)
1. **圖片資源**: 14,471 張圖片載入可能會有些慢（BGG 原圖有時很大），需要在 HTML 內為 `<img>` 設定 `loading="lazy"` 以優化滾動體驗。
2. **抽卡機率**: 修改 `startGacha()` 函數。目前是平均從所有卡池隨機撈，我們應該加入按稀有度分配的機率系統（例如：C 50%, UC 30%, R 10%, SR 7%, SSR 2.5%, UR 0.5%），並從對應陣列中抽卡。
3. **過濾空值**: 確保所有數學運算中（例如 `log(0)` 或 `min_players == 0`）有防呆機制，若有除以 0 的狀況則給予預設值。
